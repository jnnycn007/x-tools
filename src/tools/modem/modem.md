## 协议互通测试清单

以下清单用于验证本模块与常见对端实现（终端工具、BootLoader、嵌入式设备）在 xModem/yModem/zModem 下的互通性。

### 一、测试前准备

1. 串口参数一致：波特率、数据位、校验位、停止位、流控完全一致。
2. 关闭其他占用串口的软件，确认端口独占。
3. 准备测试文件集：
	- 空文件（0B）
	- 小文件（1B、127B、128B、129B）
	- 边界文件（1023B、1024B、1025B）
	- 中等文件（64KB、1MB）
	- 二进制随机文件（包含 0x00、0x1A）
4. 建议对端至少覆盖两类：
	- PC 工具链（如 lrzsz、Tera Term、SecureCRT 等）
	- 目标设备固件（BootLoader 或自研接收端）
5. 每次测试记录：协议、方向、文件名、文件大小、耗时、结果、日志关键字。

### 二、基础互通矩阵

1. xModem：Tx、Rx 各 1 轮（小文件 + 1MB 文件）。
2. yModem：Tx、Rx 各 1 轮（含文件名/文件长度元信息）。
3. zModem：Tx、Rx 各 1 轮（当前实现为调试轻量帧模式，需对端按相同帧格式）。
4. 每个协议都执行：
	- 同机回环（若可）
	- PC <-> 嵌入式设备
	- 错误注入场景（超时/取消/异常断开）

### 三、xModem 互通用例

1. 握手兼容：对端发送 C 或 NAK 时均可启动发送。
2. 块序号连续：块号与反码正确，接收端 ACK 正常推进。
3. CRC 校验：故意篡改单块，验证 NAK 与重传生效。
4. 结束流程：EOT -> ACK 流程闭环，无卡死。
5. 中止流程：任一端取消后，本端能及时退出并提示 canceled。

### 四、yModem 互通用例

1. 块 0 元信息：文件名、文件大小字段被对端正确解析。
2. 数据块传输：1K 块传输中 ACK/重传逻辑正确。
3. 结束块：EOT 后结束块（空块 0）流程被对端接受。
4. 文件一致性：传输完成后对比大小与哈希（MD5/SHA256）。
5. 多次连续传输：同一会话下连续发 3 次，验证状态复位正常。

### 五、zModem 互通用例

1. 帧头识别：对端可识别 ZPAD/ZDLE/ZDATA 头。
2. 长度字段：小端长度解析正确，不出现粘包误判。
3. CRC 校验：CRC 错误时对端返回 NAK，本端可重试或终止。
4. 结束标识：EOT 收发一致，双方会话正常结束。
5. 兼容性说明：若对端是标准 ZMODEM 实现，需确认是否支持当前调试帧格式。

### 六、异常与稳定性用例

1. 串口热拔插：传输中断开串口，界面与线程应稳定退出。
2. 超时恢复：对端无响应时应在超时后结束并输出可读错误。
3. 文件路径异常：不可写路径、只读目录、同名覆盖提示。
4. 大文件稳定性：1MB 以上文件连续 5 次传输无崩溃。
5. 并发操作防护：传输中重复点击 Start 不应启动第二个任务。

### 七、通过标准

1. 成功场景：文件大小一致且哈希一致。
2. 失败场景：出现错误时有明确日志，线程与 UI 状态可恢复。
3. 进度显示：进度不倒退，不超过 100%，结束状态正确。
4. 资源释放：停止或失败后串口句柄已释放，可立即再次启动。

### 八、日志建议关键字

1. 协议/方向：XMODEM|YMODEM|ZMODEM, TX|RX
2. 帧级信息：blockNo, retry, ack, nak, eot
3. 错误信息：timeout, crc mismatch, canceled, open port failed
4. 统计信息：file size, elapsed ms, throughput(B/s)
